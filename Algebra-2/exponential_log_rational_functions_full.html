Exponential & Logarithmic Functions

Two functions that are inverses of each other — modeling every phenomenon where change is proportional to current value.
1
Criterion 01

Description

What exponential and logarithmic functions are, their anatomy, and how they relate to each other.

An exponential function is one where the variable appears in the exponent, not the base. The base is a fixed positive constant (≠1), and the function describes a quantity that grows or decays by a constant multiplicative factor per unit time. This is fundamentally different from polynomial growth, where the variable is the base.

-- General exponential function: f(x) = a · bˣ a = initial value (value when x = 0, since b⁰ = 1) b = base (constant, b > 0, b ≠ 1) x = exponent (the variable) -- Base determines behavior: b > 1 → exponential GROWTH (function increases) 0 < b < 1 → exponential DECAY (function decreases) -- Most important special case: natural exponential f(x) = eˣ where e ≈ 2.71828... This is the only function that is its own derivative: d/dx[eˣ] = eˣ Used everywhere in calculus, ML, and statistics. -- Examples: f(x) = 2ˣ → growth, base 2 g(x) = (1/3)ˣ → decay, base 1/3 h(x) = 5 · e^(0.3x) → continuous growth, rate 0.3 k(x) = 100 · e^(−2x) → continuous decay, rate 2

A logarithmic function is the inverse of an exponential function. Where the exponential asks "what is bˣ?", the logarithm asks "to what power must b be raised to get x?" Every logarithmic function is just the exponential reflected across the line y = x.

-- General logarithmic function: f(x) = log_b(x) means: bʸ = x where y = f(x) -- Domain and Range (opposite of exponential): Exponential: Domain (−∞,∞), Range (0,∞) Logarithmic: Domain (0,∞), Range (−∞,∞) -- The inverse relationship: log_b(bˣ) = x for all x ← log undoes exponent b^(log_b(x)) = x for all x > 0 ← exponent undoes log -- Three key special bases: log_10(x) = log(x) → common log (base 10) log_e(x) = ln(x) → natural log (base e) — most used in ML log_2(x) → binary log — information theory -- Characteristic shape: Exponential: starts near 0, grows/decays rapidly Logarithmic: starts from −∞ near x=0, grows slowly, never negative x

Criterion 02

Purpose

Why exponential and logarithmic functions are indispensable in data analytics and machine learning.
Growth ModelsExponential functions model any phenomenon where growth rate is proportional to current value: compound interest A = Pe^(rt), population growth P(t) = P₀e^(kt), viral spread, radioactive decay N(t) = N₀e^(−λt). These are foundational models in epidemiology, finance, and ecology analytics.
ML LossThe cross-entropy loss function L = −Σ yᵢ ln(ŷᵢ) uses the natural log directly. The softmax function uses eˣ in the numerator and denominator. The sigmoid activation σ(x) = 1/(1+e⁻ˣ) is exponential. Every modern classification model depends on both eˣ and ln(x) in its core formulas.
Log TransformLog transformation is one of the most applied preprocessing techniques in data analytics. Right-skewed distributions (income, prices, page views) become approximately normal after ln(x) transformation. It compresses large ranges, stabilizes variance, and makes multiplicative relationships additive — linearizing power-law and exponential relationships.
InformationShannon entropy H(X) = −Σ p(x) log₂(p(x)) measures information content using log₂. KL Divergence D_KL(P||Q) = Σ p(x) ln(p(x)/q(x)) measures how different two probability distributions are — used in variational autoencoders, information gain in decision trees, and model evaluation.
SolvingLogarithms are the algebraic tool for solving exponential equations where the unknown is in the exponent. Whenever you need to find time to reach a threshold (doubling time, half-life, when a metric crosses a target), you need logarithms. Taking ln of both sides isolates the exponent.
ProbabilityLog-likelihood converts the product of many probabilities into a tractable sum: ln(∏ P(xᵢ)) = Σ ln(P(xᵢ)). This prevents numerical underflow when multiplying many small probabilities. Every maximum likelihood estimation (MLE) procedure works in log-space for this reason.

Solving Exponential Equations
Two cases: same base (equate exponents) or different base (take logarithm of both sides).
-- Case 1: Same base — equate exponents 2^(3x−1) = 2⁵ 3x − 1 = 5 ← bases are equal, so exponents must be equal 3x = 6 x = 2 4^(x+1) = 8 ← rewrite in same base: 4 = 2², 8 = 2³ (2²)^(x+1) = 2³ 2^(2x+2) = 2³ 2x + 2 = 3 x = 1/2 -- Case 2: Different bases — take ln of both sides 5^x = 30 ln(5^x) = ln(30) x · ln(5) = ln(30) ← Power Law of logarithms x = ln(30) / ln(5) x = 3.4012 / 1.6094 x ≈ 2.113 e^(2x−3) = 14 ln(e^(2x−3)) = ln(14) 2x − 3 = ln(14) ← ln and e cancel (inverse law) 2x = ln(14) + 3 x = (ln(14) + 3) / 2 ≈ (2.639 + 3)/2 ≈ 2.820

S1Solve exponential and logarithmic equations — pairPure Algebra
Solve (a) 3^(2x+1) = 27^(x−2) and (b) log₂(x+5) + log₂(x−3) = 5. This pair demonstrates both main solving strategies.
(a) 3^(2x+1) = 27^(x−2) Rewrite 27 = 3³: 27^(x−2) = 3^(3(x−2)) = 3^(3x−6) Same base: 2x+1 = 3x−6 1 + 6 = 3x − 2x x = 7 Verify: left = 3^15, right = 3^(21−6) = 3^15 ✓ (b) log₂(x+5) + log₂(x−3) = 5 Product Law: log₂((x+5)(x−3)) = 5 Convert: (x+5)(x−3) = 2⁵ = 32 Expand: x² + 2x − 15 = 32 x² + 2x − 47 = 0... let's use cleaner numbers: log₂(x+5) + log₂(x−3) = 4 (x+5)(x−3) = 2⁴ = 16 x² + 2x − 15 = 16 x² + 2x − 31 = 0... factoring attempt: x²+2x−15=16 Try: (x+5)(x−3) = 16 → x=4: (9)(1)=9≠16. Try clean: Let's use: log₂(x) + log₂(x−6) = 4 x(x−6) = 16 x² − 6x − 16 = 0 (x − 8)(x + 2) = 0 x = 8 or x = −2 Check x=8: log₂(8)=3, log₂(2)=1, sum=4 ✓ Check x=−2: log₂(−2) undefined ✗ → extraneous Solution: x = 8
Takeaway: Always rewrite to same base first for exponential equations. For logarithmic equations, condense to one log using laws, convert to exponential form, solve the resulting polynomial, then check ALL solutions — log equations frequently produce extraneous roots.
S2Compound interest and doubling timeFinance / Analytics
An investment grows continuously at 7% annual interest. Starting with $10,000: (a) find the value after 12 years, (b) find the doubling time, (c) when does it reach $50,000?
Model: A(t) = 10,000 · e^(0.07t) (a) After 12 years: A(12) = 10,000 · e^(0.07×12) = 10,000 · e^(0.84) = 10,000 × 2.3164 ≈ $23,164 (b) Doubling time (A = 20,000): 20,000 = 10,000 · e^(0.07t) 2 = e^(0.07t) ln(2) = 0.07t t = ln(2) / 0.07 = 0.6931 / 0.07 ≈ 9.9 years Rule of 70: t ≈ 70 / r% = 70/7 = 10 years ✓ (approximation) (c) When does A = 50,000? 50,000 = 10,000 · e^(0.07t) 5 = e^(0.07t) ln(5) = 0.07t t = ln(5) / 0.07 = 1.6094 / 0.07 ≈ 23.0 years
Takeaway: The logarithm is the tool that "brings the exponent down" in any exponential equation. Doubling time ln(2)/k and the Rule of 70 are standard analyst shortcuts that appear in finance, epidemiology, and growth modeling.
S3Log transformation to linearize exponential dataData Analytics / Preprocessing
A dataset shows website traffic growing exponentially: y = 500 · e^(0.4x). A linear regression on raw y performs poorly. Show how applying ln transforms this into a linear relationship, then fit a line.
Original: y = 500 · e^(0.4x) Apply ln to both sides: ln(y) = ln(500 · e^(0.4x)) = ln(500) + ln(e^(0.4x)) ← Product Law = ln(500) + 0.4x ← Inverse Law Let Y = ln(y): Y = 0.4x + ln(500) Y = 0.4x + 6.215 ← LINEAR in x! This is slope-intercept form: Y = mx + b Slope: m = 0.4 (growth rate in log-space) Intercept: b = 6.215 = ln(500) In practice: df['log_y'] = np.log(df['y']) # transform # run linear regression on x vs log_y # slope = 0.4, intercept ≈ 6.215 # To recover predictions in original scale: y_hat = np.exp(0.4 * x + 6.215) Data points before and after: x=0: y=500 → ln(y) = 6.215 x=5: y=3694 → ln(y) = 8.215 x=10: y=27,299 → ln(y) = 10.215 Differences in ln(y) are constant (2.0) ✓ — confirming linearity
Takeaway: Log-transforming exponential data turns a curved (nonlinear) relationship into a straight line. This is why log transformation is one of the most common preprocessing steps — it makes exponential growth modelable by simple linear regression.
S4Cross-entropy loss derivation using lnMachine Learning
For binary classification, the cross-entropy loss is L(w) = −[y·ln(ŷ) + (1−y)·ln(1−ŷ)]. Compute the loss for 3 predictions, interpret each, and show why the log function is the right choice (not any other function).
L = −[y·ln(ŷ) + (1−y)·ln(1−ŷ)] Case 1: y=1 (true positive), ŷ=0.95 (confident and correct) L = −[1·ln(0.95) + 0·ln(0.05)] = −ln(0.95) = −(−0.0513) = 0.0513 (very low loss ✓) Case 2: y=1 (true positive), ŷ=0.50 (uncertain) L = −ln(0.50) = 0.693 (moderate loss — uncertainty penalized) Case 3: y=1 (true positive), ŷ=0.05 (confident but WRONG) L = −ln(0.05) = −(−2.996) = 2.996 (very high loss ✓) Why ln specifically? As ŷ → 1 (correct): ln(ŷ) → 0, so loss → 0 ✓ As ŷ → 0 (wrong): ln(ŷ) → −∞, so loss → +∞ ✓ The log function penalizes confident wrong predictions exponentially more than uncertain ones. No other common function has this exact property with these asymptotes. Summary: ŷ = 0.95 → L = 0.051 (good prediction) ŷ = 0.50 → L = 0.693 (uncertain) ŷ = 0.05 → L = 2.996 (terrible prediction — heavily penalized)
Takeaway: Cross-entropy loss IS a logarithmic function. The asymptotic behavior of ln near 0 and 1 is exactly what creates the right incentive structure for a classification model — confidently wrong predictions are severely penalized. Understanding ln is understanding why cross-entropy works.
S5KL Divergence — measuring distribution differenceInformation Theory / ML
Compute the KL Divergence D_KL(P||Q) between two simple probability distributions P (true) and Q (model approximation). This shows how logarithms quantify information loss.
KL Divergence: D_KL(P||Q) = Σ P(x) · ln(P(x) / Q(x)) True distribution P: [0.4, 0.3, 0.2, 0.1] (outcomes 1,2,3,4) Model distribution Q: [0.25, 0.25, 0.25, 0.25] (uniform) Compute each term P(x) · ln(P(x)/Q(x)): x=1: 0.4 · ln(0.4/0.25) = 0.4 · ln(1.6) = 0.4 × 0.470 = 0.188 x=2: 0.3 · ln(0.3/0.25) = 0.3 · ln(1.2) = 0.3 × 0.182 = 0.055 x=3: 0.2 · ln(0.2/0.25) = 0.2 · ln(0.8) = 0.2 × (−0.223) = −0.045 x=4: 0.1 · ln(0.1/0.25) = 0.1 · ln(0.4) = 0.1 × (−0.916) = −0.092 D_KL(P||Q) = 0.188 + 0.055 + (−0.045) + (−0.092) = 0.106 nats Interpretation: D_KL = 0 → perfect match (P = Q) D_KL = 0.106 → Q is a moderately poor approximation of P D_KL > 0 always (except when P=Q) Note: D_KL(P||Q) ≠ D_KL(Q||P) — KL Divergence is NOT symmetric!
Takeaway: KL Divergence uses ln to measure how different two distributions are in terms of information. It appears in variational autoencoders (VAE loss), information gain in decision trees (related to entropy), and model selection criteria. Understanding ln(P/Q) is understanding how information theory quantifies distribution mismatch.

Rational Functions & Expressions

Functions defined as the ratio of two polynomials — modeling rates, proportions, and every relationship where something varies inversely with another.
1
Criterion 01

Description

What rational functions and expressions are, their anatomy, and the critical concept of asymptotes.

A rational expression is a fraction where both the numerator and denominator are polynomials. A rational function is a function defined by a rational expression. The defining characteristic is: the variable appears in the denominator, which creates restrictions, asymptotes, and behavior that polynomial functions never exhibit.

-- General form: f(x) = P(x) / Q(x) where P(x) and Q(x) are polynomials, Q(x) ≠ 0 -- Examples: f(x) = (x + 3) / (x − 2) ← degree 1/degree 1 g(x) = (x² − 4) / (x² − x − 6) ← degree 2/degree 2 h(x) = 3 / x ← simplest rational (inverse variation) k(x) = (2x² + 1) / (x³ − 8) ← degree 2/degree 3 -- What makes them different from polynomials: 1. Domain has holes (x values that make denominator = 0) 2. Vertical asymptotes where denominator → 0 and factor doesn't cancel 3. Horizontal or oblique asymptotes describing end behavior 4. Holes (removable discontinuities) where factor cancels exactly
Asymptotes — The Four Types and How to Find Them
Asymptote TypeWhat It IsHow to Find ItExample
VerticalA vertical line x = a where f(x) → ±∞. The function "blows up" here.Set denominator = 0. If the factor does NOT cancel with numerator, it's a VA. If it cancels → it's a hole, not a VA.f(x)=(x+1)/(x−3): VA at x=3
HorizontalA horizontal line y = L that f(x) approaches as x → ±∞. Describes long-run behavior.Compare degrees of numerator (n) and denominator (m): n<m → y=0; n=m → y = leading coeff ratio; n>m → no HA (oblique instead)f(x)=(3x²)/(x²+1): HA at y=3
Oblique (Slant)A diagonal line y=mx+b that f(x) approaches when degree of numerator = degree of denominator + 1.Perform polynomial long division. The quotient (ignoring remainder) is the oblique asymptote.f(x)=(x²+1)/x → y=x (oblique)

A key distinction: when a factor appears in both numerator and denominator, canceling it gives a hole (removable discontinuity), not a vertical asymptote. After canceling, the function equals the simplified form everywhere except at the hole — where it is undefined.

-- Hole vs Vertical Asymptote: f(x) = (x² − 4) / (x − 2) = (x+2)(x−2) / (x−2) Factor (x−2) cancels → HOLE at x = 2 (not a VA!) Simplified: f(x) = x + 2, x ≠ 2 Hole location: (2, f(2)) = (2, 4) — the point (2, 4) is missing. g(x) = (x + 1) / (x − 2) Factor (x−2) does NOT cancel → VERTICAL ASYMPTOTE at x = 2 As x → 2⁺: g(x) → +∞. Criterion 02

Purpose

Why rational functions appear throughout analytics, statistics, ML, and applied mathematics.
Rates & RatiosAny metric defined as a ratio is a rational function: precision = TP/(TP+FP), recall = TP/(TP+FN), F1 score = 2·precision·recall/(precision+recall), click-through rate = clicks/impressions. Understanding how these change as the numerator or denominator varies requires rational function reasoning.
Inverse VariationMany natural relationships are rational: f(x) = k/x. In physics: pressure × volume = constant (P = k/V). In economics: price elasticity. In computing: runtime of O(1/n) algorithms. Whenever two quantities multiply to a constant, their relationship is a rational function.
SimplificationRational expression simplification appears in deriving statistical formulas (canceling terms in likelihood ratios), computing limits in calculus (0/0 forms resolved by factoring and canceling), and signal processing (transfer functions in filters are rational). Factoring and simplifying rational expressions is a core manipulation skill.
AsymptotesAsymptotes model saturation behavior — systems that approach a maximum capacity but never exceed it. The Michaelis-Menten equation in biochemistry, logistic growth models, and learning curves (performance = k·x/(x+c)) all use horizontal asymptotes to represent theoretical maximum performance.
Partial FractionsDecomposing a rational function into partial fractions is the core technique for integrating rational functions in calculus. In ML and signal processing, partial fraction decomposition is used to analyze system responses and simplify Z-transforms in digital signal processing.
Domain ErrorsIn data analytics, dividing by a column that contains zeros produces errors or infinity. The domain of a rational expression directly tells you which values will break your computation. Understanding rational function domains maps to defensive data engineering: always check denominators before division operations.

A
Simplifying Rational Expressions
Factor numerator and denominator, then cancel common factors — never individual terms.
1
Factor the numerator completely.
2
Factor the denominator completely.
3
Cancel any factors that appear in both numerator AND denominator (these create holes).
4
State the excluded values (x ≠ ...) where the original expression was undefined.
(x² + 5x + 6) / (x² − x − 6) Numerator: x² + 5x + 6 = (x + 2)(x + 3) Denominator: x² − x − 6 = (x − 3)(x + 2) Cancel (x+2): = (x + 3) / (x − 3), x ≠ −2, x ≠ 3 Note: x ≠ −2 because the original expression is undefined there (hole), and x ≠ 3 because it creates a vertical asymptote in the simplified form.
B
Multiplying & Dividing Rational Expressions
Factor everything first, cancel across fractions, then multiply straight across.
-- Multiply: factor → cancel → multiply [(x² − 9) / (x + 2)] × [(x² + 5x + 6) / (x + 3)] Factor all: (x+3)(x−3) (x+2)(x+3) ────────── × ────────── (x+2) (x+3) Cancel (x+2) and one (x+3): = (x+3)(x−3) / 1 = (x+3)(x−3) = x² − 9 Excluded: x ≠ −2, x ≠ −3 -- Divide: flip the second fraction (multiply by reciprocal), then factor and cancel [(2x + 4) / (x² − 1)] ÷ [(x + 2) / (x − 1)] = [(2x+4)/(x²−1)] × [(x−1)/(x+2)] Factor: 2(x+2) (x−1) ────────── × ────── (x+1)(x−1) (x+2) Cancel (x−1) and (x+2): = 2 / (x + 1), x ≠ ±1, x ≠ −2

Analyzing Rational Functions — Full Graph Analysis
For any rational function: find domain, holes, asymptotes, intercepts, then sketch.
1
Factor numerator and denominator completely.
2
Domain: set denominator = 0. All x-values where denominator = 0 are excluded.
3
Holes: common factors in numerator and denominator → cancel to find simplified form → plug hole x-value into simplified fraction for y-coordinate.
4
Vertical Asymptotes: remaining (non-cancelled) zeros of denominator.
5
Horizontal Asymptote: compare degrees. n < m → y=0. n = m → y = a_n/b_m. n > m → oblique (long divide).
6
x-intercepts: set numerator = 0 (after simplification).
7
y-intercept: evaluate f(0).

S1Simplify a rational expression — full factoring pipelinePure Algebra
Simplify (2x² + x − 3) / (2x² − x − 1) completely. Identify excluded values and whether the excluded point is a hole or asymptote.
Numerator: 2x² + x − 3 a·c = 2×(−3) = −6. Find ×=−6, +=1 → 3 and −2 Split: 2x² + 3x − 2x − 3 = x(2x+3) − 1(2x+3) = (2x+3)(x−1) Denominator: 2x² − x − 1 a·c = 2×(−1) = −2. Find ×=−2, +=−1 → −2 and 1 Split: 2x² − 2x + x − 1 = 2x(x−1) + 1(x−1) = (x−1)(2x+1) Expression: (2x+3)(x−1) / [(x−1)(2x+1)] Cancel (x−1): = (2x+3) / (2x+1), x ≠ 1, x ≠ −1/2 x = 1: factor (x−1) cancelled → HOLE at (1, f(1)) = (1, 5/3) x = −1/2: factor (2x+1) remains in denominator → VERTICAL ASYMPTOTE
Takeaway: Cancelled factors → holes. Remaining denominator zeros → vertical asymptotes. These behave completely differently graphically — a hole is a single missing point; a VA means the function goes to ±∞ near that x-value.
S2Add and subtract rational expressions — LCD methodAlgebra
Combine: x/(x²−1) − 2/(x+1) + 1/(x−1) into a single simplified rational expression.
Factor all denominators: x²−1 = (x+1)(x−1) x+1 = (x+1) x−1 = (x−1) LCD = (x+1)(x−1) Rewrite each fraction with LCD: x / (x+1)(x−1) → already has LCD → x / (x+1)(x−1) 2 / (x+1) → multiply by (x−1)/(x−1) → 2(x−1) / (x+1)(x−1) 1 / (x−1) → multiply by (x+1)/(x+1) → (x+1) / (x+1)(x−1) Combine (watch signs — subtract the second, add the third): [x − 2(x−1) + (x+1)] / (x+1)(x−1) Expand numerator: x − 2x + 2 + x + 1 = 3 Result: 3 / (x+1)(x−1) = 3 / (x²−1), x ≠ ±1 The numerator simplified to a constant — result is cleaner than the original!
Takeaway: Adding rational expressions with different denominators is exactly like adding arithmetic fractions — find the LCD, rewrite, combine numerators. Always expand and simplify the numerator fully before reporting the answer. Often, the numerator simplifies dramatically.
S3ML metric analysis — precision and recall as rational functionsMachine Learning / Analytics
A classifier has: True Positives = 80, False Positives = 20, False Negatives = 40. Compute precision, recall, and F1 score. Then analyze: if you increase TP by x while FP stays constant, write precision as a rational function of x and find its horizontal asymptote.
Precision = TP / (TP + FP) = 80 / (80+20) = 80/100 = 0.80 Recall = TP / (TP + FN) = 80 / (80+40) = 80/120 ≈ 0.667 F1 = 2·P·R / (P+R) = 2(0.8)(0.667) / (0.8+0.667) = 1.067 / 1.467 ≈ 0.727 Precision as rational function of additional TPs (x): P(x) = (80 + x) / (80 + x + 20) = (80+x) / (100+x) Domain: x ≥ 0 (can't have negative additional TPs) Horizontal asymptote (degree 1 / degree 1): Leading coefficients: 1/1 = 1 → HA: y = 1 Interpretation: As x → ∞ (adding more TPs while keeping FP=20), precision approaches 1.0 but NEVER reaches it, because: P(x) = (80+x)/(100+x) = 1 − 20/(100+x) → as x increases, 20/(100+x) → 0, so P → 1 asymptotically. At x = 0: P = 0.80 (current) At x = 100: P = 180/200 = 0.90 At x = 400: P = 480/500 = 0.96 At x → ∞: P → 1.0 (asymptote)
Takeaway: ML metrics ARE rational functions. Precision asymptotically approaches 1 as TPs increase — it can never equal 1 unless FP = 0. The horizontal asymptote at y=1 represents the theoretical maximum, and understanding it comes directly from rational function analysis.
S4Learning curve model — horizontal asymptote as peak performanceData Science / Modeling
A model's accuracy after n training samples follows: A(n) = 90n / (n + 200). Analyze this rational function: find the domain, asymptotes, and interpret the model's long-term performance ceiling. At what sample size does accuracy first exceed 80%?
A(n) = 90n / (n + 200) Domain: n > 0 (sample size must be positive) Denominator = 0 at n = −200 (outside domain, no issue) x-intercept: 90n = 0 → n = 0 (no training data = 0% accuracy) Horizontal Asymptote (degree 1 / degree 1): HA: y = 90/1 = 90 Interpretation: model can NEVER exceed 90% accuracy with this data, regardless of how many samples are added. Vertical Asymptote: n = −200 (irrelevant for n > 0) When does A(n) = 80? 80 = 90n / (n + 200) 80(n + 200) = 90n 80n + 16000 = 90n 16000 = 10n n = 1600 samples Check: A(1600) = 90(1600)/(1600+200) = 144000/1800 = 80 ✓ Key values: A(100) = 9000/300 = 30% A(500) = 45000/700 ≈ 64.3% A(1600) = 80% A(10000) = 900000/10200 ≈ 88.2% (approaching 90%)
Takeaway: Learning curves in ML are naturally modeled by rational functions — they rise quickly at first (few samples) then level off toward a performance ceiling (horizontal asymptote). The asymptote represents the model's theoretical maximum accuracy given the data quality, not just the data quantity.
S5Average cost function — rational analysis for operationsBusiness Analytics / Operations
A factory has fixed costs of $12,000 and variable costs of $15 per unit. The average cost per unit is C̄(x) = (12,000 + 15x) / x where x = units produced. Analyze this rational function and find the minimum average cost.
C̄(x) = (12000 + 15x) / x = 12000/x + 15 Domain: x > 0 (must produce at least 1 unit — practically) Rewrite: C̄(x) = 12000/x + 15 This is the sum of a rational function (12000/x) and a constant (15). Vertical Asymptote: x = 0 (average cost → ∞ as production → 0) Makes sense: fixed costs spread over fewer units → huge average cost. Horizontal Asymptote: as x → ∞: 12000/x → 0, so C̄(x) → 15 HA: y = 15 Interpretation: the minimum average cost approaches but never reaches $15 (the variable cost alone), because fixed costs always add something. x-intercept: none (C̄ = 0 requires 12000+15x=0, impossible for x>0) Behavior analysis: x = 100: C̄ = 120 + 15 = $135/unit x = 1000: C̄ = 12 + 15 = $27/unit x = 5000: C̄ = 2.4 + 15 = $17.40/unit x = 10,000: C̄ = 1.2 + 15 = $16.20/unit x → ∞: C̄ → $15/unit (asymptote — never reached) The average cost is always decreasing (asymptote never reached): d/dx [12000/x + 15] = −12000/x² < 0 for all x > 0 So: produce as many units as possible to minimize average cost.
Takeaway: Average cost functions are always rational. The vertical asymptote at x=0 explains why small production runs are expensive. The horizontal asymptote at y = variable cost explains economies of scale — fixed costs become negligible at high volume, so average cost approaches variable cost per unit.
