Functions: Definition, Notation, Domain & Range

The formal mathematical structure that turns every data model into a precise, well-defined rule.
1
Criterion 01

Description

What a function is, how to read its notation, and what domain and range mean precisely.

A function is a rule that assigns to each input value exactly one output value. The critical word is exactly one — not zero outputs, not two outputs — one. This is what separates a function from a general relation.

Think of a function as a machine: you feed in an input, the machine processes it according to a fixed rule, and it produces one specific output. The same input always produces the same output — this is called determinism, and it is the same property that makes data models reliable and reproducible.

-- Formal definition: A function f from set A to set B is a rule such that every element x in A is paired with exactly one element y in B. Written: f : A → B or y = f(x) -- Reading the notation: f(x) → "f of x" — the output when input is x f(3) → "f of 3" — substitute x=3, compute the output f(x)=2x+1 → "f is the rule: multiply x by 2 and add 1" -- Examples: f(x) = 3x − 4 → f(2) = 3(2)−4 = 2 g(x) = x² + 1 → g(−3) = (−3)²+1 = 10 h(x) = √(x − 5) → h(9) = √4 = 2
⚠️
The Vertical Line Test: A graph represents a function if and only if every vertical line crosses the graph at most once. If any vertical line crosses the graph twice, it means one x-value maps to two y-values — violating the "exactly one output" rule. This is the visual definition of a function.

The domain of a function is the complete set of all valid input values — all values of x for which the function is defined and produces a real output. The range is the complete set of all possible output values — every value that f(x) can actually produce.

-- Domain restrictions arise from: 1. Square roots (or any even root): expression inside must be ≥ 0 2. Denominators: denominator must ≠ 0 3. Logarithms: argument must be > 0 4. Real-world context: e.g., time cannot be negative -- Finding domain and range: f(x) = √(x − 3) Domain: x − 3 ≥ 0 → x ≥ 3 → [3, ∞) Range: √ always ≥ 0 → [0, ∞) g(x) = 1 / (x − 2) Domain: x − 2 ≠ 0 → x ≠ 2 → (−∞, 2) ∪ (2, ∞) Range: all real numbers except 0 → (−∞, 0) ∪ (0, ∞) h(x) = x² + 1 Domain: all real numbers → (−∞, ∞) Range: x²≥0, so x²+1≥1 → [1, ∞)

Two special types worth memorizing: a one-to-one function means each output is produced by exactly one input (no two inputs share the same output) — this is needed for a function to have an inverse. A constant function maps every input to the same output: f(x) = k for all x.

Criterion 02

Purpose

Why the function concept is the most foundational idea in all of data science.
ModelingEvery machine learning model is literally a function — it maps inputs (features) to outputs (predictions). Linear regression: ŷ = f(x) = wx + b. Neural network: ŷ = f(x) = σ(Wx + b). Understanding what a function is, what its domain is, and what its range represents is the starting point for understanding every model you will ever build or use.
DomainDomain defines what inputs are valid for a model or transformation. In data preprocessing, this is critical: a log transform has domain x>0 — applying it to data with zeros or negatives will throw errors. A square root transform has domain x≥0. Domain restrictions in mathematics translate directly to data validation rules in code.
RangeRange tells you what outputs are possible. The sigmoid function has range (0,1) — this is why it works as a probability output. The softmax function has range [0,1] with all outputs summing to 1. Knowing the range tells you whether a model's output can represent what you need it to represent.
CompositionFunction composition f(g(x)) is how neural networks work — each layer applies a function to the output of the previous layer. A deep neural network with n layers is literally a chain of composed functions: f_n(f_{n-1}(...f_1(x)...)). Understanding composition is essential for understanding how information flows through a model.
InverseInverse functions undo each other: f⁻¹(f(x)) = x. The natural log is the inverse of eˣ (which you already know). In ML, the link function and its inverse are used in generalized linear models (logistic regression uses sigmoid and logit as inverses). Data transformations and their inverses are used to encode and decode predictions.
NotationFunction notation f(x) is the universal language of mathematics. Every formula in statistics (density functions, CDF, likelihood), every loss function in ML (L(w)), every model evaluation metric is written using function notation. Fluency in reading and writing it is non-negotiable.

Criterion 03

How This Works

Evaluating functions, finding domain and range, function composition, and inverses — all with worked examples.

Part A — Evaluating a Function
Substitute the input value for x everywhere in the expression. Follow order of operations precisely.

f(x) = 2x² − 3x + 1 f(4): 2(4)² − 3(4) + 1 = 2(16) − 12 + 1 = 32 − 12 + 1 = 21 f(−2): 2(−2)² − 3(−2) + 1 = 2(4) + 6 + 1 = 15 f(a+1): 2(a+1)² − 3(a+1) + 1 = 2(a²+2a+1) − 3a − 3 + 1 = 2a² + 4a + 2 − 3a − 2 = 2a² + a ← evaluating at expressions, not just numbers

Part B — Finding the Domain
Always ask: what values of x would cause a problem? The domain is everything that does NOT cause a problem.

D
Domain-Finding Algorithm
Apply to every function before using it in analytics — including data transformations.
1
Start with all real numbers (−∞, ∞) as the assumed domain.
2
Check for square roots / even roots: set the expression inside ≥ 0 and solve the inequality. Remove values that make it negative.
3
Check for denominators: set the denominator = 0 and solve. Remove those x-values.
4
Check for logarithms: set the argument > 0 and solve. Remove values that make it ≤ 0.
5
Apply all restrictions simultaneously. Write the final domain in interval notation, combining restrictions with ∩ (intersection) or expressing as a union of valid intervals.
Example — multiple restrictions at once: f(x) = √(x+4) / (x − 1) Restriction from √: x+4 ≥ 0 → x ≥ −4 Restriction from ÷: x − 1 ≠ 0 → x ≠ 1 Combined domain: x ≥ −4 AND x ≠ 1 Domain: [−4, 1) ∪ (1, ∞)

Part C — Function Composition
f(g(x)) means: first apply g to x, then apply f to the result. The output of g becomes the input of f.

f(x) = 2x + 1 g(x) = x² − 3 -- f(g(x)): Replace x in f with the entire expression g(x): f(g(x)) = 2(x² − 3) + 1 = 2x² − 6 + 1 = 2x² − 5 -- g(f(x)): Replace x in g with the entire expression f(x): g(f(x)) = (2x+1)² − 3 = 4x²+4x+1−3 = 4x²+4x−2 IMPORTANT: f(g(x)) ≠ g(f(x)) in general! Order of composition matters — just like order of operations in neural network layers.

Part D — Inverse Functions
The inverse function f⁻¹ reverses what f does. If f(a) = b, then f⁻¹(b) = a. Only one-to-one functions have inverses.

Find f⁻¹(x) for f(x) = 3x − 7: Step 1 — Replace f(x) with y: y = 3x − 7 Step 2 — Swap x and y: x = 3y − 7 Step 3 — Solve for y: x + 7 = 3y y = (x + 7)/3 Step 4 — Write as inverse: f⁻¹(x) = (x + 7)/3 Verify: f(f⁻¹(x)) = 3·((x+7)/3) − 7 = x + 7 − 7 = x ✓ f⁻¹(f(x)) = ((3x−7)+7)/3 = 3x/3 = x ✓

Samples

Five worked examples from pure algebra to ML and data analytics applications.
S1Evaluate a function at values and expressionsPure Algebra
Given f(x) = x² − 4x + 3, evaluate f(0), f(5), f(−1), and find f(x+h) — a key step in the calculus limit definition of the derivative.
f(x) = x² − 4x + 3 f(0) = (0)² − 4(0) + 3 = 0 − 0 + 3 = 3 f(5) = (5)² − 4(5) + 3 = 25 − 20 + 3 = 8 f(−1) = (−1)² − 4(−1) + 3 = 1 + 4 + 3 = 8 f(x+h) — replace every x with (x+h): = (x+h)² − 4(x+h) + 3 = x² + 2xh + h² − 4x − 4h + 3 [f(x+h) − f(x)] / h: ← this is the difference quotient = [x²+2xh+h²−4x−4h+3 − (x²−4x+3)] / h = [2xh + h² − 4h] / h = h(2x + h − 4) / h = 2x + h − 4 As h→0: limit = 2x − 4 = f'(x) ← derivative of f(x)
Takeaway: Evaluating f(x+h) and forming the difference quotient is how calculus defines the derivative. This is function notation working in full force — substituting an entire expression (x+h) instead of just a number.
S2Domain of a composite data transformation functionData Analytics
A data analyst applies the transformation T(x) = ln(√(x − 2)) to a feature column. Before running the code, they must determine the valid domain — which rows of data can be transformed without error.
T(x) = ln(√(x − 2)) Step 1 — Inner restriction (√): x − 2 ≥ 0 → x ≥ 2 Step 2 — Outer restriction (ln): argument must be > 0 √(x−2) > 0 → x − 2 > 0 → x > 2 (√(x−2) = 0 when x=2, and ln(0) is undefined) Final domain: x > 2 → (2, ∞) Also simplify T(x): T(x) = ln(√(x−2)) = ln((x−2)^(1/2)) = (1/2)·ln(x−2) Range of T: since ln(x−2) → (−∞,∞) for x>2, T(x) = (1/2)·ln(x−2) also has range (−∞, ∞) In practice: filter data before transforming: df_clean = df[df['x'] > 2].copy() df_clean['T'] = 0.5 * np.log(df_clean['x'] - 2)
Takeaway: Every data transformation is a function application, and the domain determines which values are valid. Computing the domain before transforming is equivalent to writing data validation logic — it prevents runtime errors and NaN/Inf values in your pipeline.
S3Neural network as composed functionsMachine Learning
A simple neural network with 2 layers applies: Layer 1: g(x) = ReLU(2x − 1) = max(0, 2x−1). Layer 2: f(z) = σ(3z − 0.5) = 1/(1+e^(−(3z−0.5))). Show the composed function f(g(x)), find its domain and range, and evaluate at x = 0.8.
g(x) = max(0, 2x − 1) f(z) = 1 / (1 + e^(−(3z − 0.5))) Composition: f(g(x)) = f(max(0, 2x−1)) = 1 / (1 + e^(−(3·max(0,2x−1) − 0.5))) Domain of g: all real numbers (ReLU takes any input) Range of g: [0, ∞) (ReLU outputs 0 or positive values) Domain of f(g(x)): inherits domain of g → (−∞, ∞) Range of f(g(x)): sigmoid always outputs (0, 1) Evaluate at x = 0.8: g(0.8) = max(0, 2(0.8)−1) = max(0, 0.6) = 0.6 f(0.6) = 1 / (1 + e^(−(3(0.6)−0.5))) = 1 / (1 + e^(−(1.8−0.5))) = 1 / (1 + e^(−1.3)) = 1 / (1 + 0.2725) ≈ 0.786 (78.6% predicted probability)
Takeaway: A neural network IS a composition of functions. The range of each layer is the domain of the next. Knowing the range of ReLU ([0,∞)) and sigmoid ((0,1)) tells you exactly what values flow between layers — critical for understanding vanishing gradients and output interpretation.
S4Finding and applying an inverse functionAlgebra / Preprocessing
A data analyst standardizes a feature using z-score: f(x) = (x − μ) / σ, with μ = 50 and σ = 8. Find the inverse function, which converts z-scores back to original values. Then use it to interpret a model output of z = 1.5.
f(x) = (x − 50) / 8 ← standardization (z-score) Find f⁻¹(z): Step 1: z = (x − 50) / 8 Step 2: swap x and z: x = (z − 50)/8... no, let's be precise. Let z = (x−50)/8. Solve for x: 8z = x − 50 x = 8z + 50 f⁻¹(z) = 8z + 50 ← converts z-score back to original scale Verify: f(f⁻¹(z)) = ((8z+50) − 50)/8 = 8z/8 = z ✓ f⁻¹(f(x)) = 8·((x−50)/8) + 50 = x−50+50 = x ✓ Interpret z = 1.5: x = f⁻¹(1.5) = 8(1.5) + 50 = 12 + 50 = 62 → A z-score of 1.5 corresponds to an original value of 62. → This value is 1.5 standard deviations above the mean.
Takeaway: Standardization is a function. Its inverse is the de-standardization function. When a model produces a prediction in standardized space, you apply the inverse function to recover the real-world value. This is function inversion applied to preprocessing.
S5Piecewise function — real-world threshold modelAnalytics / Modeling
A shipping cost function is defined by three tiers: free for orders under $30, flat $5 for orders $30–$100, and 5% of order value for orders over $100. Write this as a piecewise function, state its domain and range, and evaluate for orders of $20, $65, and $150.
Let x = order value in dollars. C(x) = { 0 if 0 ≤ x < 30 { 5 if 30 ≤ x ≤ 100 { 0.05x if x > 100 Domain: [0, ∞) (order value must be non-negative) Range: {0} ∪ {5} ∪ (5, ∞) = [0, ∞) — well, specifically {0, 5} ∪ (5, ∞) Simplified: [0, ∞) \ (0, 5) i.e., 0, 5, or any value >5 Evaluate: C(20) = 0 ← 20 < 30 → free shipping C(65) = 5 ← 30 ≤ 65 ≤ 100 → flat $5 C(150) = 0.05(150) = 7.50 ← 150 > 100 → 5% of order In code (Python): def shipping(x): if x < 30: return 0 elif x <= 100: return 5 else: return 0.05 * x
Takeaway: Piecewise functions model any situation where behavior changes based on input value — tax brackets, pricing tiers, activation functions (ReLU is a piecewise function!), and business rules. Writing them clearly defines exactly what happens at each threshold.

Slope and Linear Functions

The mathematics of constant-rate relationships — the single most deployed mathematical concept in all of data analytics.
1
Criterion 01

Description

What slope is, what linear functions are, and all the forms they come in.

Slope measures the rate of change of a linear relationship — how much the output (y) changes for each one-unit increase in the input (x). It is a single number that completely characterizes the direction and steepness of a line.

-- Slope formula (given two points (x₁,y₁) and (x₂,y₂)): m = (y₂ − y₁) / (x₂ − x₁) = Δy / Δx = rise / run -- Interpreting slope: m > 0 → line rises left to right (positive relationship) m < 0 → line falls left to right (negative relationship) m = 0 → horizontal line (no change in y, constant function) m → ±∞ → vertical line (undefined slope — NOT a function!) |m| → steepness: larger absolute value = steeper line Example: m = 3 → for every 1 unit increase in x, y increases by 3 m = −0.5 → for every 1 unit increase in x, y decreases by 0.5

A linear function is a function whose graph is a straight line — one where the rate of change is constant. It has the defining property: equal changes in x always produce equal changes in y. This constancy is what makes linear relationships so tractable and so widely used.

Three Forms of a Linear Equation — When to Use Each
Slope-Intercept Form
y = mx + b
Most common. m = slope, b = y-intercept. Use when you know the slope and y-intercept, or for graphing. This is also the form of every linear regression model: ŷ = w₁x + w₀.
Point-Slope Form
y − y₁ = m(x − x₁)
Best when: you know the slope and one point on the line (x₁, y₁). Most useful for constructing a line when intercept is unknown.
Standard Form
Ax + By = C
All terms on one side. A, B, C are integers with A ≥ 0. Used in systems of equations and linear programming constraints.
Two-Point Form
(y−y₁)/(y₂−y₁) = (x−x₁)/(x₂−x₁)
When only two points are given. Compute slope first, then use point-slope form. Common in interpolation.
-- Special lines: Horizontal: y = k (slope = 0, parallel to x-axis) → f(x) = k is a constant function Vertical: x = c (slope undefined, NOT a function) -- Parallel and perpendicular lines: Parallel lines: same slope m₁ = m₂, different intercepts Perpendicular lines: slopes are negative reciprocals: m₁ · m₂ = −1 e.g., m₁ = 2 → m₂ = −1/2

Criterion 02

Purpose

Why slope and linear functions are the most deployed mathematical concept in data analytics.
RegressionLinear regression is a linear function: ŷ = w₀ + w₁x₁ + w₂x₂ + ... The coefficients wᵢ ARE slopes — each wᵢ measures how much ŷ changes per unit change in xᵢ, holding all other features constant. Understanding slope = understanding what a regression coefficient means. This is the interpretability engine of linear models.
GradientIn calculus, the derivative of a function at a point is the slope of the tangent line at that point. Gradient descent — the optimization algorithm that trains every ML model — moves in the direction of steepest slope (gradient) to minimize the loss. Slope is the most fundamental concept in optimization.
ScalingMin-max normalization is a linear function: x' = (x − min) / (max − min). Z-score standardization is linear: z = (x − μ) / σ. Every linear scaling and normalization applied to features is a linear function with computable slope and intercept. Understanding this means understanding what preprocessing does to your data.
InterpolationGiven two known data points, linear interpolation uses the two-point form to estimate values in between. This is used in time series gap-filling, upsampling, and computer graphics. The slope between two points is the constant rate of change assumed between them.
CorrelationThe correlation coefficient r measures the strength and direction of a linear relationship. A positive r means positive slope; negative r means negative slope. The line of best fit has slope b₁ = r × (sᵧ/sₓ). Understanding slope is the geometric interpretation of correlation.
Cost FunctionsMany business models are linear in their cost structure: Total Cost = Fixed Cost + (Variable Cost × Units). The variable cost IS the slope. Break-even analysis, marginal analysis, and pricing optimization all rely on identifying and manipulating the slope of a cost or revenue line.

Criterion 03

How This Works

Computing slope, writing equations of lines in all forms, parallel/perpendicular, and connecting to regression.
A
Computing Slope from Two Points
Any two distinct points define exactly one line — and one slope.
1
Label the two points: (x₁, y₁) and (x₂, y₂). Order doesn't matter.
2
Compute Δy = y₂ − y₁ (change in output).
3
Compute Δx = x₂ − x₁ (change in input).
4
Divide: m = Δy / Δx. If Δx = 0, slope is undefined (vertical line).
Points: (3, 7) and (9, 1) Δy = 1 − 7 = −6 Δx = 9 − 3 = 6 m = −6/6 = −1 Interpretation: for every 1 unit increase in x, y decreases by 1.
B
Writing the Equation of a Line
From two points, or from slope and one point.
1
Compute slope m (if not given) from two points.
2
Use point-slope form: y − y₁ = m(x − x₁), using any known point (x₁, y₁).
3
Solve for y to get slope-intercept form: y = mx + b. Identify b (y-intercept).
4
Verify: substitute both original points into the final equation — both must satisfy it.
Find the line through (2, 5) and (6, 13): Step 1 — Slope: m = (13−5)/(6−2) = 8/4 = 2 Step 2 — Point-slope (using (2,5)): y − 5 = 2(x − 2) Step 3 — Slope-intercept: y = 2x − 4 + 5 y = 2x + 1 Verify (2,5): 2(2)+1 = 5 ✓ Verify (6,13): 2(6)+1 = 13 ✓
C
Parallel & Perpendicular Lines
Knowing slope relationships prevents errors in modeling and constraint formulation.
-- Parallel: same slope, different intercept Line: y = 3x + 5 Parallel through (1, 2): Same slope m = 3. y − 2 = 3(x − 1) → y = 3x − 1 -- Perpendicular: slopes multiply to −1 (negative reciprocal) Line: y = (2/3)x + 4 → m = 2/3 Perp. slope: m⊥ = −3/2 Perp. through (4, 1): y − 1 = −(3/2)(x − 4) y = −(3/2)x + 6 + 1 y = −(3/2)x + 7 Check: (2/3) × (−3/2) = −6/6 = −1 ✓
D
Slope as a Regression Coefficient — The ML Connection
This is why slope matters in data science beyond geometry.

In simple linear regression ŷ = w₀ + w₁x, the coefficient w₁ is exactly the slope of the fitted line — it tells you the expected change in y per unit change in x. The formula for computing it from data is a direct extension of the two-point slope formula:

-- Regression slope from data (OLS formula): w₁ = Σ[(xᵢ − x̄)(yᵢ − ȳ)] / Σ[(xᵢ − x̄)²] = Cov(x,y) / Var(x) -- Regression intercept: w₀ = ȳ − w₁·x̄ This is the point-slope form of the regression line through the means! If we know w₁ (slope) and the point (x̄, ȳ) lies on the line: ŷ − ȳ = w₁(x − x̄) ← exactly point-slope form -- Interpretation of w₁: w₁ = 2.5 means: holding all else equal, a 1-unit increase in x is associated with a 2.5-unit increase in predicted y. This is slope. Every regression coefficient IS a slope.

S1Find the line through two points — all three formsPure Algebra
Find the equation of the line through (−2, 3) and (4, −9). Express in slope-intercept, point-slope, and standard form.
Points: (−2, 3) and (4, −9) Slope: m = (−9 − 3) / (4 − (−2)) = −12 / 6 = −2 Point-slope form (using (−2, 3)): y − 3 = −2(x − (−2)) y − 3 = −2(x + 2) Slope-intercept form: y = −2x − 4 + 3 y = −2x − 1 Standard form (Ax + By = C, A ≥ 0): 2x + y = −1 Verify (−2, 3): −2(−2) − 1 = 4−1 = 3 ✓ Verify (4, −9): −2(4) − 1 = −8−1 = −9 ✓
Takeaway: All three forms describe the same line. Slope-intercept is best for reading slope and intercept. Point-slope is best for constructing. Standard form is best for systems of equations. Know which to reach for when.
S2Min-Max normalization — slope and intercept of scalingData Preprocessing
A feature column contains values with min = 20 and max = 80. The analyst applies min-max normalization: x' = (x − min)/(max − min). Show this is a linear function, identify its slope and y-intercept, and interpret what they mean for the data.
Min-max formula: x' = (x − 20) / (80 − 20) = (x − 20) / 60 Expand: x' = (1/60)x − 20/60 = (1/60)x − 1/3 This is a linear function: Slope: m = 1/60 ≈ 0.0167 Intercept: b = −1/3 ≈ −0.333 Interpretation: Slope: every 1-unit increase in x increases x' by 1/60 Intercept: when x = 0, x' = −1/3 (but x=0 is outside domain here) Domain: [20, 80] (original feature range) Range: [0, 1] (normalized output — guaranteed by min-max) Verify: x = 20: x' = (20−20)/60 = 0 ✓ (min maps to 0) x = 80: x' = (80−20)/60 = 1 ✓ (max maps to 1) x = 50: x' = (50−20)/60 = 30/60 = 0.5 ✓ (midpoint maps to 0.5)
Takeaway: Min-max normalization is a linear function with slope 1/(max−min) and intercept −min/(max−min). Understanding it as a linear function means you can reason about what it does to slope, intercept, relationships, and outliers — not just apply it blindly.
S3Compute regression slope and intercept from scratchLinear Regression / ML
Given 5 data points, manually compute the OLS regression line ŷ = w₀ + w₁x. This connects the slope formula directly to the regression coefficient formula.
Data: x = [1, 2, 3, 4, 5], y = [3, 5, 4, 7, 8] n = 5 Step 1 — Compute means: x̄ = (1+2+3+4+5)/5 = 15/5 = 3 ȳ = (3+5+4+7+8)/5 = 27/5 = 5.4 Step 2 — Compute Σ(xᵢ−x̄)(yᵢ−ȳ): i=1: (1−3)(3−5.4) = (−2)(−2.4) = 4.8 i=2: (2−3)(5−5.4) = (−1)(−0.4) = 0.4 i=3: (3−3)(4−5.4) = (0)(−1.4) = 0.0 i=4: (4−3)(7−5.4) = (1)(1.6) = 1.6 i=5: (5−3)(8−5.4) = (2)(2.6) = 5.2 Sum = 12.0 Step 3 — Compute Σ(xᵢ−x̄)²: (−2)²+(−1)²+0²+1²+2² = 4+1+0+1+4 = 10 Step 4 — Slope: w₁ = 12.0 / 10 = 1.2 Step 5 — Intercept: w₀ = ȳ − w₁·x̄ = 5.4 − 1.2(3) = 5.4−3.6 = 1.8 Regression line: ŷ = 1.8 + 1.2x Interpretation: each 1-unit increase in x is associated with a 1.2-unit increase in predicted y.
Takeaway: The OLS regression formula w₁ = Cov(x,y)/Var(x) is the slope formula generalized to many data points. The intercept formula w₀ = ȳ − w₁x̄ is the point-slope form applied to the centroid (x̄, ȳ). Linear regression IS slope, computed from data.
S4Linear interpolation — filling time series gapsTime Series / Data Engineering
A sensor records temperature at t=0 (18°C) and t=6 (30°C), but data is missing for t=1,2,3,4,5. Use linear interpolation to estimate the missing values.
Known points: (t=0, T=18) and (t=6, T=30) Step 1 — Slope: m = (30 − 18) / (6 − 0) = 12/6 = 2 °C per time unit Step 2 — Line equation (slope-intercept): T = 2t + 18 (b=18 since T(0) = 18 = y-intercept) Step 3 — Fill missing values: T(1) = 2(1) + 18 = 20°C T(2) = 2(2) + 18 = 22°C T(3) = 2(3) + 18 = 24°C T(4) = 2(4) + 18 = 26°C T(5) = 2(5) + 18 = 28°C T(6) = 2(6) + 18 = 30°C ✓ Python: import numpy as np t = np.arange(0, 7) T_known = {0: 18, 6: 30} T_interp = 2 * t + 18
Takeaway: Linear interpolation is literally finding a line through two known points and reading off values at intermediate x-inputs. It is the simplest gap-filling strategy in time series and uses only the slope and intercept of a line between two anchor points.
S5Marginal cost analysis — slope in business intelligenceBusiness Analytics
A manufacturer's total cost is C(x) = 4500 + 38x (dollars), where x = units produced. Revenue is R(x) = 65x. (a) Identify slope and intercept of each function and interpret. (b) Find break-even. (c) For what production range is profit increasing? (d) What is the marginal profit?
C(x) = 4500 + 38x R(x) = 65x (a) Interpret: C: slope = 38 (each unit costs $38 more to produce) intercept = 4500 (fixed costs even at x=0) R: slope = 65 (each unit sold earns $65) intercept = 0 (no revenue if nothing sold) (b) Break-even: R(x) = C(x) 65x = 4500 + 38x 27x = 4500 x = 166.67 → 167 units (round up) (c) Profit function: P(x) = R(x) − C(x) = 65x − (4500 + 38x) = 27x − 4500 P is linear with slope = 27 > 0 → Profit is ALWAYS increasing once production starts. → P > 0 when 27x > 4500, i.e., x > 166.67 (d) Marginal profit = slope of P(x) = $27 per unit Every additional unit produced earns $27 in profit.
Takeaway: In business analytics, every linear cost/revenue/profit model is a slope problem. The slope of the profit function IS the marginal profit — the economic concept of "how much extra profit per extra unit." This connects algebra slope directly to economic decision-making.
