Algebra II · Subtopic

Laws of Logarithms

The rules that govern how logarithms behave — enabling simplification, solving, and computation of expressions involving log.
📋
Criterion 01
Description

A logarithm answers the question: "To what power must we raise the base b to get the number x?"

log_b(x) = n means bⁿ = x -- Read as: "log base b of x equals n" Examples: log_2(8) = 3 because 2³ = 8 log_10(100) = 2 because 10² = 100 log_e(e²) = 2 because e² = e² (natural log, written ln)

Logarithm is the inverse of exponentiation. Just as subtraction undoes addition and division undoes multiplication, the logarithm undoes the exponential.

The Laws of Logarithms are a set of rules that allow you to manipulate, simplify, expand, and combine logarithmic expressions. They are derived entirely from the laws of exponents — every log law has a direct exponent law counterpart.

There are three special bases used constantly in data analytics:

Base 10 → log_10(x) or log(x) Common logarithm used in: pH, decibels, Richter scale, some ML metrics Base e → log_e(x) or ln(x) Natural logarithm (e ≈ 2.71828) used in: calculus, exponential models, ML loss functions Base 2 → log_2(x) Binary logarithm used in: information theory, entropy, decision trees
🎯
Criterion 02
Purpose
ComputationLogarithm laws transform multiplication into addition and division into subtraction — historically making computation of large numbers feasible (before calculators). This property still matters in computing where floating-point multiplication overflows.
ML / LossCross-entropy loss — the most used loss function in classification — is defined using natural logarithm (ln): L = −Σ yᵢ · ln(ŷᵢ). Log laws are needed to simplify, differentiate, and optimize this expression.
Info TheoryShannon entropy — the measure of information/uncertainty in a dataset — uses log base 2: H(X) = −Σ p(x) · log₂(p(x)). Decision trees use this formula to determine the best split (information gain).
StatisticsLog-likelihood is used in maximum likelihood estimation (MLE). Taking the log converts the product of probabilities into a sum — making optimization tractable: ln∏P(xᵢ) = Σ ln P(xᵢ).
ScalingLog transformation of skewed data is one of the most common preprocessing steps in analytics. It compresses large ranges, reduces skewness, and makes multiplicative relationships additive.
CalculusThe derivative of ln(x) = 1/x is one of the most used results in calculus. Log laws are prerequisites to differentiating complex expressions using logarithmic differentiation.
SolvingLogarithms are used to solve exponential equations where the unknown is in the exponent — for example, finding when a model's loss crosses a threshold, or solving compound interest for time.
⚙️
Criterion 03
How This Works

Each law is explained with its formula, the intuition behind it (derived from exponent laws), and a worked numerical example.

Law 01 Product Law
log_b(x · y) = log_b(x) + log_b(y)
The log of a product equals the sum of the logs. This comes directly from the exponent rule bᵐ × bⁿ = b^(m+n). Logarithm turns multiplication into addition.
log_2(4 × 8) = log_2(4) + log_2(8) = 2 + 3 = 5 Verify: log_2(32) = 5 because 2⁵ = 32 ✓
Law 02 Quotient Law
log_b(x / y) = log_b(x) − log_b(y)
The log of a quotient equals the difference of the logs. Derived from bᵐ ÷ bⁿ = b^(m−n). Logarithm turns division into subtraction.
log_3(81 / 9) = log_3(81) − log_3(9) = 4 − 2 = 2 Verify: log_3(9) = 2 because 3² = 9 ✓
Law 03 Power Law
log_b(xⁿ) = n · log_b(x)
The log of a power brings the exponent down as a multiplier. Derived from (bᵐ)ⁿ = b^(mn). This is the most used law in calculus and ML — it linearizes exponential relationships.
log_10(1000²) = 2 · log_10(1000) = 2 · 3 = 6 Verify: log_10(1,000,000) = 6 because 10⁶ = 1,000,000 ✓ Also works with fractions and roots: log_5(√25) = log_5(25^(1/2)) = (1/2)·log_5(25) = (1/2)·2 = 1 Verify: √25 = 5 = 5¹ → log_5(5) = 1 ✓
Law 04 Change of Base Law
log_b(x) = log_c(x) / log_c(b)
Converts a logarithm in any base to one using a convenient base (usually base 10 or base e). Essential because calculators and computers only compute log (base 10) and ln (base e) natively. This lets you evaluate any log_b(x) using those tools.
Calculate log_5(200): log_5(200) = log_10(200) / log_10(5) = 2.3010 / 0.6990 ≈ 3.292 Verify: 5^3.292 ≈ 200 ✓ In Python: import math math.log(200, 5) → 3.292... math.log(200) / math.log(5) → same result
Law 05 Identity & Inverse Laws
log_b(b) = 1     log_b(1) = 0     b^(log_b(x)) = x     log_b(bˣ) = x
These special values and inverse relationships appear constantly when simplifying. log_b(b) = 1 because b¹ = b. log_b(1) = 0 because b⁰ = 1 for any base. The inverse laws show that log and exponent are perfect inverses — they undo each other completely.
log_7(7) = 1 because 7¹ = 7 log_7(1) = 0 because 7⁰ = 1 log_7(7³) = 3 (log undoes the exponent) 7^(log_7(50)) = 50 (exponent undoes the log) ln(e⁵) = 5 (natural log undoes eˣ) e^(ln 12) = 12 (eˣ undoes natural log)

Criterion 04
Sample Problems
Sample A — Expanding a Logarithmic Expression
Expand: log_2( (8x³) / y ) Step 1 — Quotient Law: log_2(8x³) − log_2(y) Step 2 — Product Law: log_2(8) + log_2(x³) − log_2(y) Step 3 — Power Law: log_2(8) + 3·log_2(x) − log_2(y) Step 4 — Evaluate known: log_2(8) = 3 (because 2³ = 8) Result: 3 + 3·log_2(x) − log_2(y)
✔ Expand order: Quotient Law first → Product Law → Power Law → evaluate constants.
Sample B — Condensing (Combining) Log Expressions
Condense: 2·ln(x) + ln(x+1) − ln(3) Step 1 — Power Law: ln(x²) + ln(x+1) − ln(3) Step 2 — Product Law: ln(x²(x+1)) − ln(3) Step 3 — Quotient Law: ln( x²(x+1) / 3 ) Result: ln( x²(x+1) / 3 )
✔ Condense order: Power Law first → combine additions (Product) → combine subtractions (Quotient).
Sample C — Solving an Exponential Equation Using Logs
Solve: 5^(2x) = 300 Step 1 — Take ln of both sides: ln(5^(2x)) = ln(300) Step 2 — Power Law: 2x · ln(5) = ln(300) Step 3 — Divide: x = ln(300) / (2 · ln(5)) Step 4 — Evaluate: x = 5.7038 / (2 × 1.6094) = 5.7038 / 3.2189 x ≈ 1.773 Verify: 5^(2×1.773) = 5^3.546 ≈ 300 ✓
✔ Taking log of both sides is the standard technique for solving any equation where the unknown is in the exponent.
Sample D — Change of Base (Calculator Use)
Evaluate: log_6(500) Using Change of Base (base 10): log_6(500) = log_10(500) / log_10(6) = 2.6990 / 0.7782 ≈ 3.468 Using Change of Base (base e / ln): log_6(500) = ln(500) / ln(6) = 6.2146 / 1.7918 ≈ 3.468 ✓ (same answer regardless of conversion base)
✔ You can use any base for the conversion — the answer is always the same. Use ln or log₁₀, whichever is available.
Sample E — Log Transformation in Data Analytics
A dataset of house prices (in $): [80,000 · 120,000 · 450,000 · 2,500,000 · 9,800,000] Range is enormous → apply log₁₀ transformation: log(80,000) = 4.903 log(120,000) = 5.079 log(450,000) = 5.653 log(2,500,000) = 6.398 log(9,800,000) = 6.991 New range: [4.9, 7.0] ← compressed from millions to a 2-unit range.
✔ Log transformation is one of the first data preprocessing steps for skewed distributions — directly using the log function and its properties.
Sample F — Cross-Entropy Loss (ML Connection)
Binary cross-entropy loss for one prediction: L = −[y·ln(ŷ) + (1−y)·ln(1−ŷ)] Given: y = 1 (true label), ŷ = 0.85 (model's predicted probability) L = −[1·ln(0.85) + (1−1)·ln(1−0.85)] L = −[ln(0.85) + 0] L = −ln(0.85) L = −(−0.1625) L = 0.1625 Using the Inverse Law: e^(−L) = e^(ln(0.85)) = 0.85 → the loss traces directly back to the predicted probability.
✔ Cross-entropy loss is a direct application of natural log. Without log laws, you cannot derive, understand, or optimize it.

Criterion 05
Common Challenges & How to Solve Them
Thinking log(x + y) = log(x) + log(y) — one of the most common and dangerous errors in all of algebra.
Fix: The Product Law says log(x · y) = log(x) + log(y). It applies to multiplication inside the log, NOT addition. log(x + y) has NO simplification — it cannot be split. Example of the error: log(3 + 7) ≠ log(3) + log(7). Correct: log(10) = 1, but log(3) + log(7) = log(21) ≠ 1. Test yourself: always ask "is there multiplication inside the log?" before applying the Product Law.
Applying the Power Law incorrectly — writing log(x)ⁿ = n·log(x) instead of the correct log(xⁿ) = n·log(x).
Fix: The Power Law only works when the exponent is inside the log argument: log(xⁿ). If the entire log expression is raised to a power — [log(x)]ⁿ — the law does NOT apply and it cannot be simplified with this rule. [log(x)]² ≠ 2·log(x). The position of the exponent matters completely.
Forgetting the domain restrictions — taking the log of a negative number or zero.
Fix: log_b(x) is only defined when x > 0 and b > 0 and b ≠ 1. log(0) and log(−5) do not exist in real numbers. In data analytics, this appears when log-transforming data that contains zeros or negatives. The fix is to use log(x + 1) — called log1p — which shifts values so zero becomes log(1) = 0 instead of undefined. Python: np.log1p(x).
Confusing log (base 10), ln (base e), and log₂ — using the wrong base in a formula.
Fix: Each has its domain: ln is used in calculus, ML loss functions, and natural growth models. log₂ is used in information theory (entropy, decision trees). log₁₀ is used in chemistry, acoustics, and general scientific measurement. In Python: math.log(x) = ln(x). math.log10(x) = log₁₀(x). math.log2(x) = log₂(x). Always check which base a formula requires before computing.
Misapplying the Quotient Law — writing log(x) − log(y) = log(x − y).
Fix: log(x) − log(y) = log(x/y), not log(x−y). The Quotient Law converts the subtraction of logs into division inside the log. The subtraction stays outside and becomes division inside. Example: ln(10) − ln(2) = ln(10/2) = ln(5) ≈ 1.609. NOT ln(10−2) = ln(8) ≈ 2.079 — completely different values.
Not recognizing when to condense vs. expand — applying the wrong direction of a law.
Fix: Laws work in both directions. Expanding breaks one log into many (useful for simplification and differentiation). Condensing combines many logs into one (useful for solving equations and computing values). When solving an equation with multiple log terms on one side, always condense first into a single log, then remove the log by converting to exponential form: if log_b(expr) = c, then expr = bᶜ.
Trying to evaluate log without knowing the base — e.g., writing log(x) without specifying what base is meant.
Fix: In pure mathematics, "log" without a base written explicitly usually means base 10. In higher mathematics and ML literature, "log" often means the natural log (ln). In computer science and information theory, "log" often means base 2. Always clarify from context — when in doubt, write the base explicitly: log_b(x). In code, always use the specific function (math.log, math.log10, math.log2) to be unambiguous.
